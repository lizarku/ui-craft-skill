# ui-craft

**Claude Code 커스텀 스킬** — "정확하면서도 재미있는" UI를 만드는 가이드라인

> 데이터는 정확하게, 경험은 흥미롭게.
> 딱딱한 대시보드가 아니라 "다시 열어보고 싶은" 서비스를 만든다.

## 이 스킬은 뭔가요?

Claude Code로 프론트엔드 작업할 때, AI가 만든 UI는 대부분 이렇습니다:

- Inter 폰트 + 보라색 그라디언트 + 흰 배경 (일명 "AI 슬롭")
- 로딩/빈 상태/에러 처리 누락
- 접근성 무시
- 혹은 반대로 너무 딱딱하고 재미없는 결과물

**ui-craft**는 이 문제를 해결합니다. Anthropic 공식 `frontend-design` 스킬의 미학적 원칙과 커뮤니티 `baseline-ui` 스킬의 실용적 폴리싱 원칙을 **하나로 통합**하고, Vue 3 + CSS 변수 환경에 맞게 최적화했습니다.

## 핵심 특징

### 3계층 규칙 구조

```
[1순위] 프로젝트 고유 규칙 — 항상 적용
  CSS 변수 체계, 컴포넌트 구조, 타이포, 반응형, 트랜지션

[2순위] 실용 원칙 (80%) — 항상 적용
  로딩/빈/에러 상태, 접근성, 데이터 가독성, 재미 요소

[3순위] 표현 원칙 (20%) — "화려하게" 키워드로 선택적 적용
  시각적 임팩트, 그라디언트, 패럴랙스
```

### 모드 전환 — 한국어 키워드

| 모드 | 키워드 | 설명 |
|------|--------|------|
| **baseline** (기본) | `담백하게` 또는 키워드 없음 | 재미는 유지, 시각 연출은 절제 |
| **expressive** | `화려하게` | 시각적 임팩트 적극 활용 |

```
"이 컴포넌트 수정해줘"           → baseline 자동 적용
"담백하게 리포트 뷰 고쳐줘"       → baseline 명시 적용
"랜딩 페이지 화려하게 리뉴얼해줘"  → expressive 모드
```

### 디자인 철학: 흥미로움 + 질서

재미 요소는 baseline에서도 항상 적용됩니다:
- 마이크로 인터랙션 (카드 hover, 스코어 바 애니메이션)
- 색상 코딩의 감성 (단순 배경색 대신 그라디언트 힌트 + 아이콘)
- 빈 상태도 재미있게 ("아직 장비가 없어요! 첫 장비를 등록해보세요")
- 데이터에 생동감 (아이콘, 컬러, 크기 변화로 강약 표현)

단, **재미가 집중을 방해하면 안 됩니다**:
- 한 화면에 움직이는 요소 3개 이상 동시 작동 = 소음
- 장식이 데이터 가독성을 해침 = 실패
- 포인트 컬러 화면당 2개 이내

## 커스터마이징

이 스킬은 **Vue 3 + CSS 변수 + Tailwind 하이브리드** 환경용으로 만들어졌지만, 자신의 프로젝트에 맞게 수정할 수 있습니다:

### 다른 프레임워크에서 쓰려면

| 섹션 | 수정 포인트 |
|------|-----------|
| 컴포넌트 구조 | `<script setup>` → React/Svelte 패턴으로 변경 |
| CSS 변수 체계 | `--bg`, `--t1` 등을 자신의 디자인 토큰으로 교체 |
| 클래스명 접두사 | `mf-*` → 자신의 프로젝트 접두사로 변경 |
| 반응형 브레이크포인트 | 768/480/360 → 자신의 기준으로 조정 |
| Transition 표준 | duration, easing을 프로젝트 컨벤션에 맞춤 |

### 모드 키워드 변경

한국어가 아닌 환경이라면 키워드를 변경하세요:

```
담백하게 → "clean" / "minimal" / "simple"
화려하게 → "expressive" / "rich" / "vibrant"
```

## 이 스킬이 다루는 것

- CSS 변수 기반 테마 시스템
- 타이포그래피 스케일 (8단계)
- 반응형 3단계 (768px / 480px / 360px)
- Transition 표준 (hover 0.15s / 데이터 바 0.4s / 모달 0.25s)
- 로딩/빈 상태/에러 패턴
- 접근성 (aria-label, focus trap, 키보드 네비게이션)
- 한국어 텍스트 처리 (word-break, 최소 폰트 크기)
- 재미 요소 + 재미의 경계선
- QA 체크리스트 (15개 항목)

## 탄생 배경

Anthropic 공식 `frontend-design` 스킬과 커뮤니티 `baseline-ui`(ibelick/ui-skills) 스킬을 분석한 결과:

| | frontend-design | baseline-ui |
|---|---|---|
| 강점 | 미학적 방향성, AI 슬롭 방지 | 실용적 폴리싱, 접근성, 상태 관리 |
| 약점 | 기존 코드 다듬기에 부적합 | Tailwind + React 전제 |
| 충돌 | 그라디언트 권장 | 그라디언트 금지 |

둘을 **계층 구조로 통합**하여 충돌을 해결하고, 특정 프레임워크 종속성을 제거했습니다.
